ELECTRA: Pre-training Text Encoders as Discriminators Rather Than Generators
Clark, Luong, Le, Manning (2020)

Masked language modeling (MLM) pretraining methods like BERT corrupt the input by replacing tokens with [MASK] and then train the model to reconstruct the original. While effective, these approaches are inefficient because the model only learns from the small fraction of masked tokens (15%).

We propose a more sample-efficient pretraining task called Replaced Token Detection (RTD). Instead of masking input tokens, we replace them with plausible alternatives generated by a small generator network. A discriminator (ELECTRA) then predicts whether each token in the corrupted sequence is real or has been replaced.

Because the discriminator predicts on every token (not just masked ones), the model receives training signal from the entire input rather than just 15%. This makes ELECTRA significantly more compute-efficient than BERT.

Architecture: Two transformer networks trained jointly:
- Generator: Small BERT-style model that generates replacements for masked tokens using MLM
- Discriminator (ELECTRA): Full-sized model that predicts is_replaced for each token

After pretraining, only the discriminator is used for downstream tasks.

Results on GLUE benchmark (same compute as BERT):
- ELECTRA-Small: 79.9 average (vs BERT-Small 75.1)
- ELECTRA-Base: 85.1 average (vs BERT-Base 82.2)
- ELECTRA-Large: 89.7 average (vs BERT-Large 87.2)

With the same compute budget as RoBERTa-Large (trained for $2,000):
- ELECTRA-Large achieves comparable GLUE performance with 4× less compute

ELECTRA demonstrates that discriminative pretraining is substantially more compute-efficient than generative MLM pretraining.
